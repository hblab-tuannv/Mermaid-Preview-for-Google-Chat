## Summary

Reset extension version to 1.0.0, add manifest `icons` field with valid PNG placeholders, and add `npm run package` script that builds then zips `dist/` contents (entries at archive root) into a versioned zip for Chrome Web Store upload.

## What & Why

This PR implements the code+packaging changes for the first public Chrome Web Store release (Path B, v1.0.0). Three acceptance criteria are addressed:

**AC-1 — Version reset 1.0.0:** Both `public/manifest.json` and `package.json` had `"version": "1.3.0"` (internal development versions). The public first release resets to `1.0.0`. Because `package.json` is `private: true` there is no npm publish concern.

**AC-2 — Manifest `icons` + placeholder PNGs, no `action`:** Chrome Web Store recommends manifest `icons` (16/48/128). Three valid RGBA PNG placeholder icons are generated by `scripts/gen-icons.mjs` — a zero-dependency Node script using `zlib.deflateSync` (RFC 1950 zlib-wrapped DEFLATE as required by PNG spec) with hand-built IHDR/IDAT/IEND chunks and correct CRC32. Art: Google-blue (#1A73E8) background with a contrasting white minimal "M" glyph drawn from filled rectangles, scaled proportionally per size. No `action` field is added (YAGNI — toolbar popup not required).

**AC-3 — Packaging script + build-chain icons copy:** The `build` script previously ended at `copy:manifest` and did not copy icons into `dist/`. A new `copy:icons` step (`fs.cpSync` — same node `-e` convention as existing scripts) is wired in at the end of `build`. A new `package` script runs `npm run build` then `scripts/package.mjs`, which uses `archiver` v8 (`ZipArchive` class — ESM-native API) to stream `dist/` into `mermaid-preview-google-chat-v1.0.0.zip` with all entries at archive root (via `archive.directory(distPath, false)` — second arg `false` means no dest prefix). Archiver was chosen over `zip` CLI (platform dep, flattens with `-j`) per design.md §2. The output zip and `dist/` are both in `.gitignore`.

ESLint config extended with a Node globals override for `scripts/*.mjs` to allow `Buffer`.

AC-4, AC-5, AC-6 (Deployment-Plan, Store Listing assets, Runbook §7 rollback rewrite) are documentation artifacts produced in the deployment phase — deferred to release-manager.

## How to test

1. Verify version: `node -e "const p=require('./package.json');const m=require('./public/manifest.json');console.log(p.version,m.version)"` — both print `1.0.0`.
2. Verify manifest icons: `node -e "const m=require('./public/manifest.json');console.log(JSON.stringify(m.icons));console.log('action' in m)"` — shows `{"16":"icons/16.png","48":"icons/48.png","128":"icons/128.png"}` and `false`.
3. Verify PNG files: `node scripts/gen-icons.mjs` (re-runnable, deterministic) then inspect `public/icons/` — three files, valid PNG signatures, correct dimensions.
4. Verify packaging: `npm run package` — creates `mermaid-preview-google-chat-v1.0.0.zip`; unzip and confirm `manifest.json`, `content.js`, `background.js`, `icons/16.png`, `icons/48.png`, `icons/128.png` are all at archive root (not nested under `dist/`).
5. Run full suite: `npm test && npm run typecheck && npm run lint` — all pass.

## Screenshots / Notes (optional)

- AC-4/5/6 (Deployment-Plan, Store Listing assets draft, Runbook §7 rollback rewrite) are explicitly deferred to the deployment phase per task scope — release-manager produces these artifacts at Phase 6.
- `archiver` v8 is ESM-native (no default export) — `package.mjs` uses the named `ZipArchive` class directly. `archive.directory(distPath, false)` is the key call: second arg `false` disables dest prefix, placing all entries at zip root while preserving the `icons/` subdir structure.
- `gen-icons.mjs` uses `zlib.deflateSync` (RFC 1950 zlib-wrapped) — NOT `deflateRawSync` (raw DEFLATE, which would produce invalid PNG). Each scanline has one filter byte (0 = None) prepended. CRC32 uses polynomial `0xEDB88320`.
- The end-to-end AC-3 test calls `execSync('npm run package', ...)` which triggers a full Vite build (slowest step, ~1s) — `afterAll` deletes the produced zip so it is never committed.
